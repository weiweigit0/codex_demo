# 31季 Hackathon 创意：财报掘金

> 本文档作为“财报掘金”作品的补充材料，用于说明产品定位、Agent 工作流和可体验 Demo。

# 📌 创意背景

**创意来源，为什么有这个创意，解决什么问题：**

上市公司定期披露的年报、季报和招股说明书包含大量经营信息，但普通投资者、职场新人和内容创作者常常面对三重门槛：不知道去哪里找可信原始文件；看得到财务数字却不理解它们之间的关系；即使读完报告，也难以把公司经营、行业位置和风险串成一套可行动的理解。

“财报掘金”希望把“找报告、读报告、做比较、讲明白”变成一条连续工作流。用户只需输入股票代码，即可从权威披露来源获取材料，由 Agent 基于证据完成财务解读、公司画像与通俗总结。系统明确标记不确定性，不提供买卖建议，帮助用户提升理解公司的效率和质量。

**目标用户 / 使用对象：**

- 对 A 股、港美股公司有研究需求，但缺乏财务专业背景的个人用户。
- 需要快速理解公司基本面、整理财经内容的投资研究爱好者、财经内容创作者和学生。
- 需要在有限时间内完成上市公司初筛、事实核验与风险梳理的业务人员。

**使用场景：**

针对希望了解一家上市公司的普通用户，在准备研究、复盘财报或制作财经内容时，系统提供从股票代码输入、原始披露获取、证据化分析到三分钟通俗总结和视频口播脚本的一站式能力。

**现有方案以及为什么需要新的 Agent 方案：**

| 现有方案 / 做法 | 优点 | 问题 / 不足 | 新 Agent 方案可以带来哪些变化 |
| - | - | - | - |
| 财经网站的行情页和财务指标页 | 获取关键指标快 | 指标碎片化，缺少原始披露上下文与解释 | 将指标、原始报告片段和分析结论关联展示，并保留证据来源 |
| 手工下载年报、季报后自行阅读 | 信息完整、来源权威 | 文档长、术语密集、跨期对比成本高 | 自动获取并整理报告，按报告期构建可检索的证据上下文 |
| 通用大模型直接问答 | 交互自然、上手门槛低 | 容易依赖模型记忆，可能产生无证据结论 | 以统一知识整理层和引用证据约束 Agent 输出，缺失信息明确提示 |
| 人工制作公司解读或短视频脚本 | 表达通俗、适合传播 | 资料收集和核验耗时，难规模化 | 基于已验证数据生成三分钟总结、问答与分镜建议，保留人工复核空间 |

# 🎯 解决方案

**解决方案简介：**

财报掘金是面向普通用户的 AI 财报与公司研究助手。系统支持 A 股与美股公司查询：从 SEC、巨潮资讯等披露源获取财报和公告，将 PDF、网页及百科补充资料标准化为本地知识数据；再由 DeepSeek 驱动的财报分析 Agent、公司画像 Agent 和三分钟财报总结 Agent 分别完成不同任务。产品主页负责公司入口，财报掘金、公司画像、三分钟总结和短视频脚本页面独立运行，仅通过股票代码与统一知识整理层协作。

**核心功能 / 价值：**

- **权威资料自动获取与本地沉淀：** 支持从 SEC、美股披露资料及巨潮资讯等来源获取公开报告；基础资料按公司和报告期存储，优先复用本地知识，缺失时再更新。
- **财报掘金分析：** 支持选择单年、单季、多年或多季度，Agent 对收入、利润、现金流、资产负债和风险事实进行证据化解读；美股英文材料也统一输出为简体中文。
- **一页看懂公司：** 读取最近三年年报与招股说明书等材料，形成主营业务、发展历程、关键人物、行业位置、资本动作与非财务风险的公司画像；资料缺失时依次使用维基百科、百度百科补充，并标记来源。
- **三分钟财报总结：** 以普通人可理解的语言给出综合评价、评分卡、关键看点、风险和观察项，支持围绕结果继续追问。
- **短视频脚本与分镜：** 将总结结果改写为口播稿、分段时长与可实现的画面建议，服务财经科普创作。
- **支持公司查询中心：** 用户可查看当前可覆盖的公司和市场范围，降低“输入后无可用报告”的不确定性。

**用户使用流程：**

1. 用户注册并登录主页，输入股票代码或从头部公司中选择目标公司。
2. 用户选择“开始财报分析”“一页看懂公司”或“三分钟财报总结”。
3. 系统解析市场与公司标识，从统一知识整理层读取财务数据、报告文本块和已有模型结果；数据不足时触发披露源更新。
4. 财报分析 Agent 先抽取带证据引用的经营与风险事实，再完成趋势、利润质量、现金流和资产负债解读；公司画像与总结 Agent 使用各自独立编排流程。
5. 用户在页面查看结论、证据、不确定性和风险关注项；可针对三分钟总结继续提问，或跳转生成短视频口播稿与分镜。

**作品亮点：**

1. **不是“问公司”，而是“读披露后再回答”：** Agent 的核心输入是经过标准化整理的财务事实和报告证据块，结论需引用真实证据；无证据内容不会作为确定事实输出。
2. **面向普通人的多种表达：** 同一份研究基础数据可服务深度财报页、公司画像、一页总结和短视频脚本，降低财务知识的阅读门槛。
3. **解耦的产品与数据架构：** 各业务页面独立演进，统一知识整理层承担数据获取、PDF 文本化、标准化与缓存，避免每个页面重复联网和解析。

# 🤖 Agent / AI 能力说明

## 1. 使用了哪些 AI / Agent 能力？

- **大模型理解与生成：** 使用 DeepSeek 对财报、报告片段和百科内容进行中文事实抽取、分析、总结、问答与脚本生成。
- **RAG / 知识库检索：** 统一知识整理层保存标准化文档、证据块、已验证财务事实、百科资料和模型产物；Agent 优先检索这些本地资料。
- **多阶段 Agent 编排：** 财报 Agent 按“事实抽取 → 趋势解读 → 风险等级评估”分阶段运行；公司画像将风险事实与风险评估拆分；三分钟总结独立汇聚多类证据。
- **工具调用与数据分析：** 后端按市场调用 SEC、巨潮资讯、百科来源及文档处理能力；对多报告期指标进行比较和质量校验。
- **缓存与自动化更新：** 原始数据与模型结果均可在本地数据库复用，并支持按季度更新、后台预热或手动刷新。
- **人机协同：** 用户选择公司与报告期、阅读证据和不确定性、提出追问；系统负责重复的资料收集、整理与表达工作。

## 2. Agent 在作品中承担什么角色？

| Agent 角色 | 是否使用 | 说明 |
| - | - | - |
| 助手 | 是 | 帮助用户查找可用公司、理解财报指标、生成通俗总结并回答追问 |
| 分析师 | 是 | 基于已验证财务事实、报告证据和公司资料，提炼趋势、经营变化和风险关注项 |
| 执行者 | 是 | 触发披露资料获取、文档标准化、知识检索、缓存读取与模型任务执行 |
| 协作者 | 是 | 将专业披露转为可阅读的结论，同时保留证据与不确定性，供用户判断和复核 |
| 流程编排者 | 是 | 为财报分析、公司画像、三分钟总结和视频脚本维护独立编排与异步任务状态 |

## 3. 和普通 AI 问答相比，新在哪里？

财报掘金不是把股票代码直接交给模型生成答案，而是构建了“数据获取—标准化—证据检索—Agent 分阶段分析—结果缓存—用户追问”的完整闭环。

- **连续任务：** 输入股票代码后，系统自动完成公司识别、报告期选择、数据校验、材料准备和异步分析。
- **证据优先：** 财务事实与报告文本块有稳定标识，模型输出必须关联有效证据；没有足够材料时会显示不确定性或资料缺失。
- **降低幻觉：** 风险事实与风险等级分离；模型提示词禁止补充未披露的数字、原因或行业事实；输出语言统一为简体中文，英文资料只作为理解依据。
- **多页面独立协作：** 财报分析、公司画像与三分钟总结互不绑定，均可使用统一知识整理层与已验证数据，既能复用结果，又便于单独迭代。
- **可持续运行：** 数据源和模型产物具备本地缓存、版本和失效机制，而不是每次访问都重新抓取网页或调用模型。

# 🎬 作品 Demo

## 1. 可体验链接

- **访问链接：** [https://www.accountstatement.top](https://www.accountstatement.top)
- **登录方式：** 首次访问可在注册页创建用户名、密码和手机号；登录后进入产品主页。
- **体验步骤：**
  1. 在主页输入 A 股或美股股票代码，例如 `000001`、`300750` 或 `AAPL`。
  2. 点击“立即体验”进入财报掘金服务页，选择年报、季报或多期对比后发起分析。
  3. 返回主页，可选择“一页看懂公司”生成公司画像，或进入“三分钟财报总结”查看通俗解读、评分卡和追问能力。
  4. 在三分钟总结页点击短视频脚本入口，查看口播稿与分镜建议。
- **注意事项：** 本作品用于公开财报信息的学习、研究和内容整理，不构成任何投资建议。首次查询或报告资料缺失时，系统可能需要等待数据更新和 Agent 任务完成。

# 📍 预期价值

**对用户：** 将理解一家公司的起点从“下载并通读数百页报告”缩短为“输入股票代码并阅读证据化结论”，同时保留回到原始证据和不确定性提示的能力，帮助普通用户建立更可靠的财报阅读习惯。

**对内容与研究流程：** 统一沉淀公开披露数据、文档文本和模型产物，减少重复抓取、重复解析和重复总结；研究者与内容创作者可以在财报解读、公司画像和短视频脚本之间复用同一份知识基础。

**对平台能力：** 项目已经形成可扩展的单机正式演示架构：统一知识整理层与各业务页面解耦，支持后续扩展更多市场、报告类型、行业对比、人工审核和团队协作能力，同时保持稳定的公网访问地址。
