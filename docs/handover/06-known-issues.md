# 已知问题与验收清单

## 已知问题

| 优先级 | 问题 | 现状与排查方向 |
| --- | --- | --- |
| 高 | 财报 Agent 的前端错误提示过于泛化 | `financial_agent/orchestrator.py` 的 `_public_error` 会合并多类异常。排查时应看任务记录、`model_runs.error_message`、应用日志和外部数据源状态。 |
| 高 | 外部数据源与模型均可能超时 | SEC、巨潮、百科和模型服务都有网络、频率和代理依赖。不能将失败直接归因于“公司不支持”。 |
| 高 | A 股 PDF 质量差异明显 | 扫描 PDF、表格列错位与单位误判可能使事实质量校验失败；必须保留失败而非由模型补造数字。 |
| 中 | 大上下文模型分析存在时间与成本压力 | 多年报告会扩大证据上下文。应控制块数、利用输入指纹缓存，并观察任务耗时。 |
| 中 | 公司画像任务数据归属尚未做鉴权 | 这是已知暂缓项。对外开放前必须补充用户/任务权限校验。 |
| 中 | 单机演示版不具备高可用 | SQLite、本地卷和后台线程适用于单机演示。多实例前需迁移 PostgreSQL、对象存储和队列。 |
| 低 | 旧文档可能仍描述早期规则分析 | 以本交接包、当前代码和 `docs/data_platform.md` 为准；改动功能后同步修订架构文档。 |

## 每次改动前

1. 阅读相关领域的路由、编排器和数据层调用，不跨域直接复用内部实现。
2. 检查 `git status`，保留用户已有的未提交改动。
3. 识别是否会影响提示词版本、缓存指纹、数据迁移或生产环境变量。

## 每次改动后

1. 运行关联单元测试；跨 Agent、视频或数据层改动时运行完整测试集。
2. 运行 `python -m compileall -q backend`。
3. 在浏览器验证页面没有布局溢出、按钮失效或轮询卡死。
4. 至少执行 A 股 `000001`、美股 `AAPL`、美股 `BIDU` 的关键路径冒烟测试。
5. 对模型改动检查：简体中文输出、事实与推断分离、证据引用、无投资建议、证据不足时显式降级。
6. 对数据层改动检查：缓存命中、强制刷新、外部失败降级和旧缓存不被失败结果覆盖。
