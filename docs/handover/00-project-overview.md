# 项目概览

## 产品定位

“财报掘金”面向非专业用户，将上市公司披露文件、结构化财务数据和受证据约束的模型分析转化为可理解的公司研究内容。系统不提供买卖建议、价格预测或投资评级。

## 用户侧能力

| 能力 | 页面 | 核心输入 | 核心输出 |
| --- | --- | --- | --- |
| 登录与主页 | `app/login.html`、`app/register.html`、`app/home.html` | 用户名、密码、手机号、股票代码 | 会话、导航、代码与市场参数 |
| 财报掘金 | `app/index.html` | 股票代码、市场、报告期 | 财务趋势、风险、问答、证据约束的 DeepSeek 解读 |
| 公司画像 | `app/profile.html` | 股票代码、市场 | 最近三年年报与可得招股书上的公司画像和非财务风险 |
| 查询中心 | `app/support.html` | 市场、公司名称或代码 | 当前数据源支持范围 |
| 3 分钟财报总结 | `app/summary.html` | 股票代码、市场、报告期 | 通俗总结、经营理解分、可追问问答 |
| 短视频脚本 | `app/summary-video.html` | 总结 ID | 口播稿、分镜与可签名的视频生产简报 |
| 音视频生产 | `media_app/` | 已签名简报 | 管理员审批后的 TTS、视频任务与 FFmpeg 成片 |

## 典型用户路径

1. 用户注册或登录后进入主页。
2. 用户输入股票代码并选择市场。
3. 主页只通过 URL 参数跳转到财报掘金、公司画像或 3 分钟财报总结；主页不承载分析逻辑。
4. 业务模块从统一知识整理层读取数据；缺失时由数据层更新公开数据源并缓存。
5. 视频脚本通过签名简报导入独立媒体服务；媒体服务不读取主系统的数据库或模型密钥。

## 当前市场与数据源

- 美股：SEC EDGAR 公司检索、XBRL 财务事实、10-K/20-F 与可得招股书 HTML。
- A 股：巨潮资讯公告索引、PDF 与文本提取。
- 公司基础背景：Wikipedia 优先，失败时百度百科；均仅作为披露资料的补充来源。
- 模型：默认通过 OpenAI 兼容接口接入 DeepSeek，当前默认模型由 `DEEPSEEK_MODEL` 配置。

## 重要术语

- **统一知识整理层**：`backend/data_platform/`。统一管理公司、披露、原始文件、文本、证据块、财务事实和模型缓存。
- **证据块**：从一份披露文档中生成、可追溯到文档与页码/章节的文本片段。
- **已验证财务事实**：经数据质量规则检查后保存的结构化指标，模型不能将其外的数据伪装成事实。
- **模型运行指纹**：由公司、报告版本、上下文和 Agent 版本构成的缓存键；相同输入优先命中缓存。
