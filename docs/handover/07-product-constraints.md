# 产品与技术约束

## 页面与领域边界

- 首页是入口和导航层，只传递 `ticker`、`market` 等查询参数，不实现财报分析、画像、总结或视频生产。
- 财报掘金、公司画像、查询中心、3 分钟总结和音视频生产是独立模块；除明确 API、URL 跳转和标准化知识外，不建立隐式依赖。
- 查询中心仅返回主页，不应重新耦合财报掘金页面。
- 音视频服务独立部署、独立环境变量、独立存储；唯一输入是经过签名的 Video Brief。

## 数据与模型边界

- 数据平台是外部基础数据的唯一入口。业务域优先读取本地知识，缺失时请求数据平台更新。
- 正式披露优先级高于百科；百科是补充且同样需要由模型按提示词处理。
- 模型可以解释、归纳、标记不确定性，但不能发明财务数字、管理层履历、竞争格局或风险事实。
- 面向用户的模型自由文本必须为简体中文；不能因为美股披露为英文而输出英文报告。
- 模型输出必须是 JSON 并经过本地校验；风险事实抽取与风险等级评估保持两阶段分离。

## 合规与安全边界

- 产品必须展示研究辅助与非投资建议性质，不提供买卖建议、目标价或确定性收益承诺。
- 密钥、电话号码、密码散列、会话、披露原文和缓存均不得进入 Git。
- 密码必须只保存哈希；生产环境使用 HTTPS。
- 视频生成必须先经管理员审批；演示模式不得伪造真实外部视频已生成的状态。
